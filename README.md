
# 17-01
Структура репозитория

history/ — хранение историй изменений
market/ — приложения для продуктов, корзины и их категорий
users/ — регистрация, аутентификация пользователей
Основные возможности

Создание продуктов и категорий вручную
Автоматическое создание профиля при регистрации
Создание истории изменений
Регистрация и аутефикация
Изменения профиля
Экспорт данных в формате CSV
Быстрый API (основные конечные точки)

GET market/cart/ — получить все корзины

GET market/cart/id — просмотр конкретной корзины

PUT market/cart/id — обновить корзину

GET market/category/ — просмотр категорий

POST market/category/ — создание категории

GET market/category/id — просмотр отдельной категории

PUT market/category/id - обновление информации про категорию

GET market/export/product - экспорт данных продуктов

GET market/products/ - вывод всех существующих продуктов

POST market/products/ - создание продукта

GET market/products/id - вывод конкретного продукта

PUT market/products/id - изменение информации про продукт

DELETE market/products/id - удаление продукта

POST users/user/register - регистрация пользователя

POST users/user/login - аутификация пользоватееля

GET users/user/logout - выход пользователя

DELETE users/user/deleto/int:pk — удаление пользователя

POST users/profile/ — создание профиля

GET users/profile/ — получить все профили пользователей

GET users/profile/int:pk — получение профиля конкретного пользователя

PUT users/profile/int:pk — обновить информацию в профиле пользователя

DELETE users/profile/int:pk — удалить профиль

Модели (ключевые поля)
Пользователь {идентификатор, имя пользователя, пароль, адрес электронной почты, имя, фамилия}
Профиль{ id, пользователь, имя, фамилия }
Категория {название, подкатегория}
Товар {название, цена, описание, наличие}
Корзина {владелец}
CartItem {товар, корзина, количество}
HistoryUserApp {пользователь, статус, дата создания}
Сигналы
post_save, User, Cart — после сохранения пользователя автоматически создается корзина для него
post_save, User, Profile — после создания/обновления пользователя создается его профиль
pre_delete, User, HistoryUserApp — после удаления пользователя в историю записывается уведомление
Экспорт
GET market/export/product — экспорт данных о продуктах в формате CSV
